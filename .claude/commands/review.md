# Code Review 命令

请对当前更改进行代码审查，重点关注以下方面：

## 审查范围
- 分析 `git diff` 中的所有更改

## 审查要点

### 1. 代码质量
- 代码是否清晰易读
- 命名是否符合项目规范（组件 PascalCase，函数 camelCase）
- 是否有重复代码可以抽取

### 2. TypeScript 类型
- 类型定义是否完整
- 是否避免使用 `any`
- 接口/类型是否合理

### 3. React 最佳实践
- Hooks 使用是否正确（依赖数组）
- 是否有不必要的重渲染
- 组件拆分是否合理

### 4. 安全性
- 是否有 XSS 风险
- 敏感信息是否暴露
- API 调用是否安全

### 5. 性能
- 是否有明显的性能问题
- 大列表是否需要虚拟滚动
- 是否有内存泄漏风险

### 6. 项目规范
- 是否符合 ESLint 规则
- 是否符合 Prettier 格式
- 国际化文案是否完整

## 输出格式

请按以下格式输出审查结果：

```
## 总体评价
[简要总结代码质量]

## 发现的问题
### 🔴 严重问题
- [问题描述及建议]

### 🟡 建议改进
- [问题描述及建议]

### 🟢 优点
- [做得好的地方]

## 修改建议
[具体的代码修改建议]
```
